IDEA又双叒叕崩溃了，怎么去找崩溃原因

二月 02, 2019
  • IDEA 崩溃
  • Scala

IDEA 你又挂了

更新了Jetbrain新的IDEA，用来写Scala玩。然后令人痛不欲生的事情发生了.
刚写了不到3行代码

我以为是偶发现象，于是杀掉了进程重启了IDEA

然后更过分的事情：

说实话，我从来没有经历过CPU 1000%以上的时候，这一血被IDEA夺了。

然后我回退了版本，2018.3.3，依旧不行。写不了几行CPU风扇就蹭蹭转，IDEA假死。
只能上网找原因，搜了半小时都没结果

那么重点来了，你起码得找到哪里出问题了，才能搞定假死的问题吧。

面向搜索引擎编程失败之后，只能面向日志编程了。

首先杀死IDEA，重启。然后一个字不动。立刻打开日志

然后回到IDEA继续写代码，等IDEA假死之后，立刻打开日志（双击就可以，Apple自带的日志浏览器还是很棒的）
打开之后一堆报错！

Scala 原来是你！

Scala plugin看来兼容3.4不是很好。去Plugins 删掉这个拖后腿的渣渣。然后查查有没有新的
去plugins网站找找有没有在3.4发布后更新的插件

有一个2019的，我下载下来发现安装不了，这货是给新的2019.1 EAP使用的。机智的我点了一下

重新安装新版本的Scala插件，虽然不是稳定版，但是CPU终于不狂飙了！

ps：2019-02-01 今天又更新了新的Scala插件 stable版

总结经验

  1. 任何IDE崩溃都是可以也应该去看崩溃日志的。
  2. 崩溃日志可以搜索错误，或者寻找出错的地方。一般都能查到原因
  3. 升级新版本的软件时，要备份好旧版本，以备不时之需。
  4. 新版本的软件如果有配套插件需要升级，务必及时升级。

最后祝看官大爷们春节快乐
